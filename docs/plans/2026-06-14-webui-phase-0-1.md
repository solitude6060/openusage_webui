# WebUI Phase 0-1 Plan

## Scope

Implement the first local-only WebUI skeleton and manual usage workflow from the WebUI fork spec.

## Branch Flow

- Base branch: `main`
- Delivery branch: `dev`
- Merge gate: triple review plus fresh verification

## Phase 0 Deliverables

- Create `apps/server`.
- Create `apps/web`.
- Create `packages/core`.
- Create `packages/storage`.
- Create `packages/providers`.
- Add root WebUI scripts.
- Implement health endpoint.
- Implement SQLite init under `~/.openusage-webui/openusage.sqlite`.
- Add `README_WEBUI.md`.

## Phase 1 Deliverables

- Implement core usage/provider types.
- Implement SQLite schema and storage API.
- Implement provider status operations.
- Implement manual usage creation.
- Implement MiniMax manual settings.
- Implement providers, usage records, and usage summary endpoints.
- Build minimal dashboard, providers, sessions, and settings pages.

## Docs To Update

- `README_WEBUI.md`
- `README_WEBUI.zh-TW.md`

- `bun install`
- `bun run test:webui`
- `bun run build:webui`
- Local smoke test on `127.0.0.1:6736`

## Deferred
